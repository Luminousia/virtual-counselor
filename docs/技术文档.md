## 项目概述

### 产品定位

本项目是一款基于 Web 的虚拟数字人心理咨询应用。用户可以与名为"小暖"的 3D 虚拟形象进行自然语言对话，获得温暖、专业的心理支持。

### 核心特性

| 特性 | 描述 |
|------|------|
| 🎭 **3D 虚拟形象** | 基于 VRM 格式的高质量 3D 模型渲染 |
| 💬 **流式对话** | 实时流式 AI 响应，边生成边显示 |
| 🔊 **语音合成** | 使用 Genie-TTS 实现高质量中文语音 |
| 👄 **口型同步** | 基于音频频谱分析的实时口型同步 |
| 😊 **表情控制** | 根据对话情感自动切换表情 |
| ✨ **自然动画** | 自动眨眼、注视跟随等微动作 |

### 角色设定

**小暖** - 虚拟数字人心理咨询师

- **温和 (Gentle)**：用温和、柔和的语调与来访者交流
- **耐心 (Patient)**：耐心倾听，给予充分时间思考和表达
- **热情 (Enthusiastic)**：对每一位来访者充满真诚的关怀

## 核心模块

### 1. 流式 AI 服务 (StreamingAIService)

**文件**: `src/services/ai/streamingAIService.ts`

实现与 DeepSeek API 的流式通信，支持边生成边输出。

```typescript
// 核心方法
async streamResponse(
  userMessage: string,
  onChunk: (chunk: string) => void,    // 每个文本块的回调
  onComplete: (fullText: string) => void // 完成时的回调
): Promise<void>
```

**特性**：
- 使用 `stream: true` 启用 SSE 流式输出
- 维护对话历史（最近 20 条）
- 内置系统提示词定义角色人设

### 2. 分句器 (SentenceSplitter)

**文件**: `src/services/ai/sentenceSplitter.ts`

将流式文本分割为可朗读的句子单元，实现"边生成边播放"。

```typescript
// 分割规则
- 主要分隔符：。！？.!?（句子结束）
- 次要分隔符：，；,;（长句分割，超过50字符时触发）
- 最小句子长度：5 字符
```

### 3. TTS 队列管理器 (TTSQueueManager)

**文件**: `src/services/tts/ttsQueueManager.ts`

管理语音合成和播放队列，实现流畅的语音输出。

```typescript
// 队列状态
interface AudioChunk {
  text: string
  audio: ArrayBuffer | null
  status: 'pending' | 'generating' | 'ready' | 'playing' | 'done' | 'error'
}
```

**工作流程**：
1. 接收句子 → 加入队列
2. 异步生成音频（调用 Genie-TTS API）
3. 按顺序播放（边生成边播放）
4. 实时分析音频频谱 → 驱动口型同步

### 4. 口型同步分析器 (LipSyncAnalyzer)

**文件**: `src/utils/lipSyncAnalyzer.ts`

基于音频频谱分析实现 5 元音口型同步。

```typescript
// 输出结构
interface LipSyncResult {
  A: number  // 张大嘴 (低频主导)
  E: number  // 微张嘴 (中频主导)
  I: number  // 扁嘴   (中高频)
  O: number  // 圆嘴   (低频主导)
  U: number  // 嘟嘴   (高频主导)
  volume: number // 音量
}
```

**算法特点**：
- 基于 FFT 频谱分析
- Winner + Runner 策略（只混合最大的两个口型）
- 平滑过渡（指数衰减）
- 静音检测（防止嘴巴抖动）

### 5. 表情控制器 (ExpressionController)

**文件**: `src/features/emoteController/expressionController.ts`

管理 VRM 模型的表情状态和平滑过渡。

```typescript
// 支持的情感状态
'neutral'   // 中性（嘴巴微张）
'happy'     // 开心
'sad'       // 悲伤
'angry'     // 生气
'surprised' // 惊讶
'thinking'  // 思考
```

**特性**：
- 平滑过渡动画（缓动函数）
- 自动眨眼 (AutoBlink)
- 注视跟随 (AutoLookAt)
- 多口型混合

### 6. VRM 模型渲染 (VRMModel)

**文件**: `src/components/VirtualHuman/VRMModel.tsx`

Three.js + @pixiv/three-vrm 实现 3D 模型渲染。

**功能**：
- 加载 VRM 格式模型
- 设置自然姿势 (A-Pose)
- 根据情感设置表情
- 实时口型同步
- 根据文本类型触发动作（问句→思考，问候→开心等）

### 7. 情感分析器 (EmotionAnalyzer)

**文件**: `src/services/ai/emotionAnalyzer.ts`

分析文本情感和类型，驱动表情和动作。

```typescript
// 情感检测（关键词匹配）
happy: ['开心', '高兴', '快乐', '幸福', ...]
sad: ['难过', '悲伤', '伤心', ...]
angry: ['生气', '愤怒', ...]
surprised: ['惊讶', '意外', ...]
thinking: ['思考', '想想', ...]

// 文本类型检测
'question'  // 问句（？结尾）
'emphasis'  // 强调（！结尾）
'greeting'  // 问候（你好、早上好等）
'agreement' // 同意（好的、明白等）
'normal'    // 普通
```

### 完整对话流程

```
用户输入文本
     │
     ▼
┌─────────────────┐
│ ChatWindow.tsx  │ ─────────────────────────────────────┐
└────────┬────────┘                                      │
         │ 发送消息                                       │
         ▼                                               │
┌─────────────────────┐                                  │
│ StreamingAIService  │                                  │
│   调用 DeepSeek API │                                  │
└────────┬────────────┘                                  │
         │ 流式返回文本块                                  │
         ▼                                               │
┌─────────────────┐                                      │
│ SentenceSplitter │ ← 分割为句子                         │
└────────┬────────┘                                      │
         │ 完整句子                                       │
         ▼                                               │
┌─────────────────┐      ┌──────────────────┐            │
│ TTSQueueManager │ ────►│ Genie-TTS Server │            │
│   管理TTS队列    │      │  生成音频(WAV)    │            │
└────────┬────────┘      └──────────────────┘            │
         │ 播放音频                                       │
         ▼                                               │
┌─────────────────┐                                      │
│ LipSyncAnalyzer │ ← 分析频谱                           │
└────────┬────────┘                                      │
         │ 口型数据 (A/E/I/O/U)                          │
         ▼                                               │
┌─────────────────────────────────────────┐              │
│           VRMModel.tsx                   │              │
│  ┌─────────────────────────────────┐    │              │
│  │    ExpressionController         │    │◄─────────────┘
│  │  - 应用口型 blendshape          │    │   情感分析结果
│  │  - 设置表情                     │    │   (完成时)
│  │  - 自动眨眼/注视                │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

```

     │
```

## API 接口

### Genie-TTS Server API

#### 健康检查

```http
GET /health
```

**响应**：
```json
{
  "status": "healthy",
  "genie_available": true,
  "loaded_characters": ["feibi"]
}
```

#### 获取角色列表

```http
GET /characters
```

**响应**：
```json
{
  "characters": ["mika", "feibi", "thirtyseven"],
  "loaded": ["feibi"],
  "default": "feibi"
}
```

#### 语音合成

```http
POST /v1/audio/speech
Content-Type: application/json

{
  "text": "你好，我是小暖",
  "character": "feibi",
  "language": "zh-CN",
  "speed": 1.0
}
```

**响应**：
- Content-Type: `audio/wav`
- Body: WAV 音频二进制数据

### DeepSeek API

```http
POST https://api.deepseek.com/v1/chat/completions
Authorization: Bearer {API_KEY}
Content-Type: application/json

{
  "model": "deepseek-chat",
  "messages": [...],
  "temperature": 0.7,
  "max_tokens": 500,
  "stream": true
}
```

**响应** (SSE 格式)：
```
data: {"choices":[{"delta":{"content":"你好"}}]}
data: {"choices":[{"delta":{"content":"，"}}]}
data: {"choices":[{"delta":{"content":"我是小暖"}}]}
data: [DONE]
```

## 部署指南

### 环境要求

| 组件 | 要求 |
|------|------|
| Node.js | 18+ |
| Python | 3.10+ |
| 内存 | 8GB+ (Genie-TTS 模型需要) |
| 显卡 | 可选，CPU 也可运行 |

### 安装步骤

#### 1. 前端

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

#### 2. Genie-TTS 服务器

```bash
cd genie-tts-server

# 创建虚拟环境
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # Linux/Mac

# 安装依赖
pip install -r requirements.txt

# 启动服务器
python server.py
```

**首次运行说明**：
- Genie-TTS 首次运行会自动下载模型数据（约 391MB）
- 默认加载 `feibi` 角色（中文女声）

### 启动顺序

1. 启动 Genie-TTS 服务器 (`localhost:8000`)
2. 启动前端开发服务器 (`localhost:5173`)
3. 浏览器访问 `http://localhost:5173`

## 已知限制

1. **口型同步精度**：基于频谱分析的口型同步是近似算法，无法达到专业动画的精度
2. **TTS 延迟**：Genie-TTS 生成音频需要时间，长句子可能有明显延迟
3. **模型兼容性**：目前仅测试了特定 VRM 模型，其他模型可能需要调整
4. **浏览器兼容**：需要支持 WebGL 和 Web Audio API 的现代浏览器
