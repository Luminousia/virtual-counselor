## 一、项目分析

### 1.1 AIRI 项目特点

| 特性 | 描述 |
|------|------|
| **架构** | Rust 后端 + TypeScript/Vue.js 前端 |
| **模型支持** | VRM + Live2D 双格式支持 |
| **动画系统** | 自动眨眼、视线追踪、情感表情、手势动作 |
| **口型同步** | 实时音频分析 + 音素映射 + 多 blendshape 混合 |
| **TTS** | 支持流式输出、边生成边播放 |
| **状态管理** | 完善的角色状态机，支持多种情感状态切换 |

**AIRI 口型同步核心实现：**
- 使用 FFT 分析音频频率
- 将频率映射到 5 个元音口型（A, E, I, O, U）
- 支持口型权重混合，实现自然过渡
- 平滑插值算法，避免口型跳变

### 1.2 ChatVRM 项目特点

| 特性 | 描述 |
|------|------|
| **架构** | Next.js + TypeScript |
| **模型支持** | VRM 格式，使用 @pixiv/three-vrm |
| **TTS** | 支持 ElevenLabs、Koeiro、VOICEVOX 等多种服务 |
| **流式处理** | AI 响应流式输出 + TTS 分段生成 |
| **表情系统** | 情感标签驱动表情变化 |
| **动画** | 程序化待机动画 + 说话时动作增强 |

**ChatVRM TTS 优化策略：**
- AI 响应分句流式输出
- TTS 分段生成，边生成边播放
- 音频预加载队列
- 支持中断和续播

### 1.3 两者对比

| 对比项 | AIRI | ChatVRM | 当前项目 |
|--------|------|---------|----------|
| **TTS 响应** | 流式 | 流式 | ❌ 等待完整生成 |
| **口型精度** | 5 元音混合 | 音量驱动 | ⚠️ 基础音量 |
| **表情系统** | 多状态机 | 情感标签 | ⚠️ 简单切换 |
| **动作系统** | 丰富手势 | 基础待机 | ⚠️ 简单呼吸 |
| **语境感知** | AI 分析 | 关键词 | ❌ 无 |

## 二、当前问题

### 2.1 TTS 响应慢（核心问题）

**问题描述：**
- 当前流程：AI 生成完整回复 → TTS 生成完整音频 → 开始播放
- 用户等待时间：AI 响应时间 + TTS 生成时间（通常 3-10 秒）

**根本原因：**
```
用户输入 → [等待 AI] → 完整回复 → [等待 TTS] → 完整音频 → 播放
           ↑___________ 串行等待，延迟叠加 ___________↑
```

**期望流程：**
```
用户输入 → AI 流式输出 → 分句 → TTS 分段生成 → 边生成边播放
           ↑__________ 并行处理，大幅减少等待 __________↑
```

### 2.2 口型同步不精确

**问题描述：**
- 当前仅使用音量驱动单一口型（aa）
- 缺少元音识别，口型单调
- 口型变化生硬，缺少平滑过渡

**现有代码问题（lipSyncAnalyzer.ts）：**
```typescript
// 当前：简单的频率阈值判断
if (lowFreqEnergy > midFreqEnergy && lowFreqEnergy > highFreqEnergy) {
  result.A = volume * 0.8  // 缺少精确的元音识别
}
```

### 2.3 动作单一，缺乏语境感知

**问题描述：**
- 只有基础的呼吸和轻微摆动
- 说话时没有手势动作
- 情感表达依赖简单关键词匹配

**期望功能：**
- 根据对话内容触发相应手势（点头、摇头、招手等）
- 情感分析驱动表情变化
- 说话时增加自然的身体动作

### 2.4 模型处理效率

**问题描述：**
- 模型加载时间较长
- 缺少模型预加载和缓存机制
- 部分 VRM 模型姿势异常（T-pose 问题）

```typescript
// 新增：流式 AI 服务
class StreamingAIService {
  async *streamResponse(message: string): AsyncGenerator<string> {
    const response = await fetch(apiUrl, {
      method: 'POST',
      body: JSON.stringify({ stream: true, messages: [...] }),
    })
    
    const reader = response.body.getReader()
    const decoder = new TextDecoder()
    
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      yield decoder.decode(value)
    }
  }
}

// 新增：分句器
class SentenceSplitter {
  private buffer = ''
  
  feed(chunk: string): string[] {
    this.buffer += chunk
    const sentences: string[] = []
    
    // 按标点分割
    const regex = /([^。！？，；\n]+[。！？，；\n])/g
    let match
    while ((match = regex.exec(this.buffer)) !== null) {
      sentences.push(match[1])
    }
    
    // 保留未完成的部分
    this.buffer = this.buffer.slice(regex.lastIndex)
    return sentences
  }
}

// 新增：TTS 队列管理器
class TTSQueueManager {
  private queue: string[] = []
  private isProcessing = false
  private audioQueue: ArrayBuffer[] = []
  
  async addSentence(sentence: string) {
    this.queue.push(sentence)
    if (!this.isProcessing) {
      this.processQueue()
    }
  }
  
  private async processQueue() {
    this.isProcessing = true
    
    while (this.queue.length > 0) {
      const sentence = this.queue.shift()!
      const audio = await this.generateTTS(sentence)
      this.audioQueue.push(audio)
      this.playNext()
    }
    
    this.isProcessing = false
  }
}
```

```typescript
// 改进的口型分析器
class AdvancedLipSyncAnalyzer {
  private analyser: AnalyserNode
  private frequencyData: Float32Array
  private smoothedWeights = { A: 0, E: 0, I: 0, O: 0, U: 0 }
  private smoothingFactor = 0.3 // 平滑系数
  
  constructor(audioContext: AudioContext) {
    this.analyser = audioContext.createAnalyser()
    this.analyser.fftSize = 2048
    this.frequencyData = new Float32Array(this.analyser.frequencyBinCount)
  }
  
  update(): LipSyncResult {
    this.analyser.getFloatFrequencyData(this.frequencyData)
    
    // 计算各频段能量
    const bands = this.calculateFrequencyBands()
    
    // 映射到元音权重
    const rawWeights = this.mapToVowels(bands)
    
    // 平滑插值
    for (const key of ['A', 'E', 'I', 'O', 'U'] as const) {
      this.smoothedWeights[key] = this.lerp(
        this.smoothedWeights[key],
        rawWeights[key],
        this.smoothingFactor
      )
    }
    
    return { ...this.smoothedWeights, volume: bands.total }
  }
  
  private calculateFrequencyBands() {
    const sampleRate = this.analyser.context.sampleRate
    const binSize = sampleRate / this.analyser.fftSize
    
    return {
      low: this.getEnergyInRange(200, 400, binSize),    // A
      midLow: this.getEnergyInRange(400, 800, binSize), // O
      mid: this.getEnergyInRange(800, 1200, binSize),   // E
      midHigh: this.getEnergyInRange(1200, 2000, binSize), // I
      high: this.getEnergyInRange(2000, 4000, binSize), // U
      total: this.getEnergyInRange(100, 4000, binSize),
    }
  }
  
  private mapToVowels(bands: FrequencyBands) {
    const total = bands.low + bands.midLow + bands.mid + bands.midHigh + bands.high
    if (total < 0.01) return { A: 0, E: 0, I: 0, O: 0, U: 0 }
    
    return {
      A: Math.min(1, bands.low / total * 2),
      O: Math.min(1, bands.midLow / total * 2),
      E: Math.min(1, bands.mid / total * 2),
      I: Math.min(1, bands.midHigh / total * 2),
      U: Math.min(1, bands.high / total * 2),
    }
  }
  
  private lerp(a: number, b: number, t: number) {
    return a + (b - a) * t
  }
}
```
